## File Setup

### 1. Create `docker-compose.yml`

Save this in `/Users/hakim/holeeMonth/ONDEWEI-LARAVEL-HAKIM/docker-compose.yml`:

```yaml
version: '3.8'

services:
  # MySQL Database
  mysql:
    image: mysql:8.0
    container_name: ondw-mysql
    environment:
      MYSQL_ROOT_PASSWORD: root_password
      MYSQL_DATABASE: ondewei_local
      MYSQL_USER: ondewei_user
      MYSQL_PASSWORD: user_password
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./database:/docker-entrypoint-initdb.d  # Auto-run SQL files on startup
    networks:
      - ondw-network
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      timeout: 20s
      retries: 10

  # Redis Cache
  redis:
    image: redis:7-alpine
    container_name: ondw-redis
    ports:
      - "6379:6379"
    networks:
      - ondw-network
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      timeout: 10s
      retries: 5

  # PHP/Laravel Application
  app:
    build:
      context: .
      dockerfile: Dockerfile.local  # See Dockerfile section below
    container_name: ondw-app
    working_dir: /app
    environment:
      - APP_ENV=local
      - APP_DEBUG=true
      - APP_URL=http://localhost:8000
      - DB_CONNECTION=mysql
      - DB_HOST=mysql
      - DB_PORT=3306
      - DB_DATABASE=ondewei_local
      - DB_USERNAME=ondewei_user
      - DB_PASSWORD=user_password
      - CACHE_DRIVER=redis
      - REDIS_HOST=redis
      - REDIS_PORT=6379
      - QUEUE_CONNECTION=redis
      - SESSION_DRIVER=redis
    ports:
      - "8000:8000"
    volumes:
      - .:/app
      - ./storage:/app/storage
      - ./bootstrap/cache:/app/bootstrap/cache
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy
    networks:
      - ondw-network
    command: >
      sh -c "
        composer install &&
        php artisan key:generate &&
        php artisan migrate --force &&
        php artisan serve --host=0.0.0.0 --port=8000
      "

volumes:
  mysql_data:

networks:
  ondw-network:
    driver: bridge
```

### 2. Create `Dockerfile.local`

Save this in `/Users/hakim/holeeMonth/ONDEWEI-LARAVEL-HAKIM/Dockerfile.local`:

```dockerfile
FROM php:8.1-cli

# Install dependencies
RUN apt-get update && apt-get install -y \
    git \
    curl \
    zip \
    unzip \
    nodejs \
    npm \
    libpng-dev \
    libjpeg-dev \
    libfreetype6-dev \
    libonig-dev \
    libxml2-dev \
    && docker-php-ext-install -j$(nproc) \
        gd \
        pdo_mysql \
        mbstring \
        xml \
        redis \
    && apt-get clean

# Install Composer
RUN curl -sS https://getcomposer.org/installer | php -- --install-dir=/usr/local/bin --filename=composer

WORKDIR /app

# Copy existing code
COPY . .

# Install PHP dependencies
RUN composer install --no-interaction --no-ansi

# Build frontend assets
RUN npm install && npm run build

EXPOSE 8000

CMD ["php", "artisan", "serve", "--host=0.0.0.0", "--port=8000"]
```

## Getting Started

```bash

# Inside container:
php artisan tinker

>>> Cache::get('test');  # Should work (Redis)
>>> exit
```

## Testing Service Worker + Push Notifications

### Local Testing Checklist

1. **Access the app**
   - Visit http://localhost:8000
   - Login with test account

2. **Test Service Worker**
   - Open DevTools → Application → Service Workers
   - Should show "Service Worker" registered
   - Hard refresh: Cmd+Shift+R
   - Check console for SW registration messages

3. **Test Push Notifications API**
   - Open DevTools → Network
   - Check if VAPID key endpoint works:
     ```
     GET http://localhost:8000/api/push/vapid
     # Should return: {"publicKey": "..."}
     ```

4. **Test Subscription**
   - Navigate to admin panel (where push notification test component is)
   - Click "Enable Notifications"
   - Browser should ask for permission
   - Verify subscription created (check browser Console)

5. **Test Sending Notification**
   - In Laravel console, send test notification:
     ```bash
     docker exec -it ondw-app bash
     php artisan tinker
     >>> Mail::raw('test', function($m) { $m->to('test@example.com'); });
     ```
   - Or use the admin panel send button if available

## Troubleshooting

### Issue: Port 8000 already in use
```bash
# Change port in docker-compose.yml
ports:
  - "8001:8000"  # Use 8001 instead

# Access at http://localhost:8001
```

### Issue: MySQL connection failed
```bash
# Verify MySQL is healthy
docker-compose ps mysql

# Check MySQL logs
docker-compose logs mysql

# Re-init database
docker exec ondw-mysql bash -c "mysql -uroot -proot_password ondewei_local < /docker-entrypoint-initdb.d/your_file.sql"
```

### Issue: Redis connection failed
```bash
# Verify Redis is running
docker exec ondw-redis redis-cli ping
# Should return: PONG
```

### Issue: Service Worker not registering
```bash
# Check browser console for errors
# May need to clear cache:
docker exec ondw-app php artisan cache:clear
# Hard refresh browser: Cmd+Shift+R
```

### Issue: CORS errors
```bash
# Check CORS config in Laravel
# May need to add localhost to cors_allowed_origins in config/cors.php
```

## After Testing: Deploy to Hostinger Staging

Once everything works locally:

1. **Commit changes to GitHub**
   ```bash
   git add .
   git commit -m "feat: Uncomment service worker and add push notifications (tested in Docker pre-prod)"
   git push origin main
   ```

2. **Deploy to Hostinger staging.onewei.my**
   - SSH to Hostinger
   - Pull latest code: `git pull origin main`
   - Run migrations: `php artisan migrate --force`
   - Restart: `php artisan cache:clear`

3. **Test on Hostinger staging**
   - Visit https://staging.onewei.my
   - Repeat Service Worker + Push Notification tests

4. **Deploy to Production**
   - After staging tests pass
   - SSH to production server
   - Pull & migrate
   - Clear cache
